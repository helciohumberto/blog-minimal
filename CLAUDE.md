## Architecture

### Static export for GitHub Pages

`next.config.ts` sets `output: "export"` — this generates a fully static site into `out/`. The `basePath` is `/blog-minimal` in production and empty in dev. Any absolute image `src` paths in rendered Markdown are rewritten at parse time in `lib/posts.ts` to prepend the basePath.

**Important:** Because this is a static export, route handlers must use `export const dynamic = "force-static"` (see `app/rss.xml/route.ts`). No server-side runtime is available in production.

### Post data layer (`lib/posts.ts`)

All content lives in `posts/*.md`. The frontmatter schema is defined in `types/post.ts`:

```
title, date (YYYY-MM-DD), excerpt, category, tags[], readingTime (auto-calculated)
```

Valid categories are a fixed union: `"estudos" | "blockchain" | "tecnologia" | "criptomoedas"`. Adding a new category requires updating both `types/post.ts` and the `VALID_CATEGORIES` array in `app/categorias/[categoria]/page.tsx`.

`lib/posts.ts` exports:
- `getAllPosts()` — all posts sorted by date desc (returns `PostMeta[]`, no HTML)
- `getPost(slug)` — single post with rendered HTML (async, uses remark pipeline)
- `getPostsByCategory(category)` — filtered by category

The remark pipeline: `remark-gfm → remark-rehype → rehype-highlight → rehype-stringify`. Syntax highlighting uses highlight.js classes; the theme is custom CSS in `globals.css`.

### Routing

| Route | File | Notes |
|---|---|---|
| `/` | `app/page.tsx` | Lists all posts via `SearchBar` |
| `/posts/[slug]` | `app/posts/[slug]/page.tsx` | Static params from `getAllPostSlugs()` |
| `/categorias/[categoria]` | `app/categorias/[categoria]/page.tsx` | Static params from `VALID_CATEGORIES` |
| `/rss.xml` | `app/rss.xml/route.ts` | Force-static RSS feed |
| `/sitemap.xml` | `app/sitemap.ts` | Next.js sitemap convention |

### Client-side search

`SearchBar` is the only `"use client"` component. It receives the full `PostMeta[]` list from the server (passed as a prop from `app/page.tsx`) and runs fuzzy search via Fuse.js on `title`, `excerpt`, `tags`, and `category`.

### Styling

All styling uses inline styles + a small set of CSS classes in `globals.css`. There is no Tailwind utility usage in components — Tailwind is imported in `globals.css` via `@import "tailwindcss"` but the project only uses its reset/base layer and the custom CSS variables. Light/dark theme is toggled by adding/removing the `.dark` class on `<html>`. The `ConstellationBg` component renders animated stars as absolutely-positioned `<div>` elements via inline styles injected into the DOM on mount.

### OG images

Static PNG files in `public/og/<slug>.png`. Generated by `scripts/gen-og.mjs` using `sharp`. When adding a new post, add its entry to `gen-og.mjs` and re-run the script to produce the PNG. The `app/posts/[slug]/page.tsx` metadata references `${SITE_URL}/og/${slug}.png`.

### Deployment

Push to `main` triggers `.github/workflows/deploy.yml`, which runs `npm ci && npm run build` and deploys `out/` to GitHub Pages. Live URL: `https://helciohumberto.github.io/blog-minimal`.
